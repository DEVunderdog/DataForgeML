# Held-back evaluation for ImputationFitDiagnostic R² — re-fit final model on all data

When computing `r2_train` for `ImputationFitDiagnostic`, we hold back 20% of complete rows before fitting the model, evaluate R² on those rows, then re-fit the final stored model on all available complete rows. The alternative — computing R² on the same rows used for training (in-sample) — was rejected because in-sample R² is optimistic: a model that memorised training data would score high even if it generalises poorly, defeating the purpose of the diagnostic. The extra re-fit is a one-time cost at pipeline setup time (not at inference), and the resulting R² is an honest estimate of how well the model predicts values it has never seen. Changing to in-sample evaluation later would silently alter the meaning of all existing `r2_train` values in serialised `FittedImputer` records.
